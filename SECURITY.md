# Security Policy

## Supported Versions

Security fixes are best-effort during the current pre-release phase.

## Reporting a Vulnerability

Please do not open public GitHub issues for security problems.

Instead:

- Use GitHub's private vulnerability reporting flow if it is enabled for the repository
- Or contact the maintainer privately through an existing trusted channel

When reporting a vulnerability, include:

- A short description of the issue
- Steps to reproduce
- Impact assessment if known
- Any suggested remediation or workaround

We will try to acknowledge reports promptly and coordinate a responsible fix and disclosure timeline.
